App Concept

Use Case C (Your Own). The app solves the problem of tracking Magic: The Gathering cards encountered in the wild. It uses a hybrid approach: Online for identification (Scryfall API) and Offline (SQLite) for collection management.

Target User

Traders

Need to quickly log card prices and details.

Players

Bookmark cards seen during gameplay for later.

Usability Test Plan

Method: Remote moderated testing with 5 participants.
Hypothesis: "Users can scan, verify, and categorize a card in under 10 seconds without external instruction."

Research Questions

  • Efficiency: How quickly can the OCR detect the card name?
  • Discoverability: Is the "Swipe-to-Delete" gesture intuitive without a visible trash button?
  • Feedback: Does the user know if the camera is actively searching?
View Heuristic Evaluation (Initial Prototype)

Visibility of System Status

Negative: The Camera preview is static. Users commented they weren't sure if the app was "frozen" or actively OCR-scanning. Needs a loading indicator.

Match between System & World

Negative: The concept of "Favorites" vs "Categories" was initially confusing. Users expected a folder system like a physical binder.

Help and Documentation

Critical: The Swipe-to-Delete function is a "Hidden Affordance." There is no visual cue (like a trash icon) until the user starts swiping. This makes the feature hard to discover.

User Control (Error Prevention)

Negative: Deleting a card happens instantly on swipe. A confirmation dialog or "Undo" snackbar is missing to prevent accidental data loss.
